# Hanzi Brush 中文版 UX 设计

## 1. 产品定位

Hanzi Brush 是一款面向美国 App Store 的 iPhone 汉字书法与中国文化 App。它不是普通中文字典，也不是只有图片的文化百科，而是一个可每天打开、可练、可收藏、可生成作品的本地工具。

核心体验：

- 每天认识一个有文化含义的汉字。
- 观看笔顺和结构提示。
- 在手机上临摹练习。
- 生成可保存、分享的书法卡片。
- 用本地记录保存练习进度、收藏和作品。

一句话卖点：

Learn the beauty of Chinese characters through guided brush practice and cultural stories.

## 2. 目标用户

主要用户：

- 美国用户，对中国文化、中文、书法、春节、东方审美感兴趣。
- 学中文的初学者，希望用更轻松的方式认识汉字。
- 喜欢制作壁纸、贺卡、社交分享图的人。

次要用户：

- 海外华人家庭，希望孩子认识汉字文化。
- 日语用户，对汉字、书道、中国文化有兴趣。
- 设计、纹身、茶文化、冥想、东方艺术爱好者。

## 3. 产品命名与品牌

英文名：Hanzi Brush  
中文名：墨韵汉字  
日文名：漢字ブラシ  
Bundle ID：com.zhouyajie.hanzibrush

品牌气质：

- 安静、清晰、有文化感。
- 面向美国市场时不堆砌古风元素，避免像“资料库”。
- 界面保持浅色宣纸质感，使用墨色、朱砂红、青绿色作为识别色。

## 4. 信息架构

底部五个主入口：

- 今日：每日汉字、连续学习天数、继续练习、最近作品。
- 练习：选择汉字、看笔顺、临摹、保存练习记录。
- 作品：选择模板，生成卡片、壁纸、贺卡，保存到本地作品集。
- 字库：按主题浏览汉字，搜索，收藏，查看学习状态。
- 设置：语言、购买状态、隐私政策、支持页面、邮件联系、数据管理。

设置页不展示 Bundle ID。

## 5. 核心功能闭环

### 5.1 每日一字闭环

用户路径：

1. 用户打开 App，进入“今日”。
2. 看到当天汉字、拼音、英文/中文/日文含义、文化说明。
3. 点击“开始临摹”进入练习。
4. 完成至少一次描摹后点击“保存练习”。
5. 返回“今日”后，连续天数、今日完成状态、最近练习卡片立即更新。

验收标准：

- 今日入口清楚，首屏即可看到当天汉字。
- 未完成时按钮显示“开始临摹”；完成后显示“再次练习”。
- 保存后回到首页必须显示今日已完成。
- 连续天数、完成数量、最近记录都保存在本地。
- 关闭 App 再打开，完成状态仍存在。

### 5.2 临摹练习闭环

用户路径：

1. 用户从“今日”或“字库”进入某个汉字详情。
2. 点击“笔顺演示”观看动画或分步提示。
3. 在临摹画布上跟随格线练习。
4. 可撤销、清空、重写。
5. 点击“保存练习”。
6. 系统保存练习次数、最后练习时间、草稿缩略图或练习摘要。
7. 返回字库或首页后，汉字卡片显示“已练习”状态。

验收标准：

- 每个入口进入练习页后都能返回来源页面。
- 撤销、清空、保存按钮有效。
- 没有笔迹时保存按钮禁用，并显示深色可读的禁用文字。
- 保存后出现明确反馈，例如“已保存到练习记录”。
- `@Published` 数组/字典更新时必须重新赋值或显式发送变化，确保 UI 刷新。

### 5.3 作品生成闭环

用户路径：

1. 用户进入“作品”。
2. 选择一个汉字或从已练习汉字继续。
3. 选择模板：宣纸、朱砂、夜墨、节日。
4. 调整文字、印章、背景、署名。
5. 点击“生成作品”。
6. 保存到本地作品集，可调用系统分享或保存到相册。
7. 返回“作品”列表后看到新作品缩略图。

验收标准：

- 模板入口清楚，至少提供 4 种模板。
- 生成按钮在没有选择汉字时不可用。
- 生成后作品进入本地作品列表。
- 作品列表为空时显示明确空状态和创建入口。
- 删除作品后列表立即刷新，并且本地数据同步删除。

### 5.4 字库与收藏闭环

用户路径：

1. 用户进入“字库”。
2. 可按主题筛选：祝福、自然、情感、季节、生肖。
3. 可搜索汉字、拼音、英文含义。
4. 点击汉字卡片进入详情。
5. 点击收藏按钮。
6. 返回字库后，收藏状态立即显示。
7. 进入收藏筛选，可看到刚收藏的字。

验收标准：

- 搜索、筛选、收藏状态组合后结果正确。
- 收藏按钮有明确选中和未选中状态。
- 收藏状态保存在本地。
- 空结果页提供返回全部字库的入口。

### 5.5 设置与支持闭环

用户路径：

1. 用户进入“设置”。
2. 可选择语言：English、简体中文、日本語。
3. 选择后 App 文案立即切换，并持久保存。
4. 未选择语言时，App 根据系统/地区推断，无法识别时使用英文。
5. 点击隐私政策或支持页面，打开 GitHub Pages 网页。
6. 点击邮件支持，打开系统邮件发送给 jay212315@gmail.com。

验收标准：

- 设置页不显示 Bundle ID。
- 语言选择保存后重启仍生效。
- 网页入口只在用户点击后打开，不主动发起网络请求。
- 支持邮箱清楚可见。
- 隐私政策和支持网页后续必须提供英语、简体中文、日语版本。

## 6. 本地数据设计

内置只读数据：

- 汉字基础资料：字、拼音、英文含义、中文解释、日文解释、主题、笔画数、部首。
- 笔顺数据：本地 JSON 或 SVG 路径。
- 文化说明：短文本，按三语言本地化。
- 模板配置：背景、文字颜色、印章位置、默认排版。

用户本地数据：

- 显式选择的语言。
- 是否完成首次引导。
- 收藏汉字集合。
- 每个汉字的练习次数、最后练习时间、今日完成状态。
- 连续学习天数。
- 生成作品的元数据和本地图片路径。
- 购买/解锁状态。

推荐实现：

- 小型设置：`UserDefaults` 或 `@AppStorage`。
- 结构化用户记录：SwiftData 或本地 JSON 文件。
- 作品图片：App Documents 目录。
- 所有本地数据更新统一通过 store 方法完成，避免视图直接改嵌套字典导致 SwiftUI 不刷新。

## 7. 截图专用假数据模式

目标：

App Store 截图需要漂亮、稳定、可重复的数据，但正式启动和 Release 版本不能出现任何假数据。

设计原则：

- 只允许通过显式启动参数启用，例如 `--screenshot-demo-data`。
- 只在 DEBUG 或专用 Screenshot 构建配置中编译假数据代码。
- Release 编译时假数据 provider 不参与编译或直接返回不可用。
- 假数据不写入生产存储目录。
- UI 不根据“是否没有数据”自动注入假数据。
- 截图脚本负责启动模拟器、传入参数、清理临时容器、生成截图。
- 发版前校验脚本检查 Release 配置中不存在截图启动参数和 demo data 注入。

建议实现边界：

- `ProductionDataProvider`：读取内置汉字数据和用户本地数据。
- `ScreenshotDemoDataProvider`：仅 DEBUG 可用。
- `AppLaunchConfiguration`：解析启动参数，但 Release 中忽略截图参数。
- `scripts/capture_screenshots.sh`：只用于截图。
- `scripts/preflight_release_check.sh`：发版前校验。

验收标准：

- 正常启动没有任何演示作品、演示收藏、演示进度。
- 带 `--screenshot-demo-data` 的 Debug 启动显示稳定演示数据。
- 退出截图模式后，正式容器没有残留假数据。
- Release 构建中截图参数无效且不显示假数据。

## 8. 可读性与审核安全

界面策略：

- App 采用浅色设计，强制浅色模式。
- Info.plist 设置 `UIUserInterfaceStyle` 为 `Light`。
- SwiftUI 根视图设置 `.preferredColorScheme(.light)`。
- 所有浅底文字使用显式深色，不依赖系统动态颜色。
- placeholder、禁用按钮、说明文字都使用足够深的颜色。

建议颜色：

- 主文字：`#1C1712`
- 次级文字：`#5F5348`
- 说明文字：`#6F6258`
- 禁用文字：`#82786E`
- 浅底背景：`#F8F1E6`
- 页面背景：`#EFE4D2`
- 主按钮：`#A9362B`
- 主按钮文字：`#FFF8EE`
- 辅助色：`#0F6F66`
- 分割线：`#D8C8B4`

验收标准：

- iOS 深色模式下界面仍保持浅色。
- 所有文字在白色、米色、浅纸色背景上可读。
- 禁用状态也不能浅到看不清。
- App Store 审核截图中无白字贴白底问题。

## 9. 付费设计建议

建议商业模式：

- 免费下载，内购解锁 Pro。
- 免费内容：每日一字、10 个基础汉字、2 个模板、基础练习。
- Pro 内容：完整主题字库、高级模板、节日作品包、无数量限制作品保存。

付费闭环：

1. 用户触达 Pro 字或高级模板。
2. 看到清晰的锁定状态和价值说明。
3. 点击解锁进入系统内购。
4. 购买成功后当前页面立即解锁。
5. 恢复购买入口在设置页可见。

第一版如果想降低复杂度，也可以先做付费下载 App，暂不接内购。这样实现更简单，但转化测试空间更小。
